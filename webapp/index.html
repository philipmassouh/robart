FAQ:

What is Robart?

Robart is a Robotic Access and Retrieval System. In other words, it is an AI-powered warehouse management system that can help you retrieve warehouse stock using natural human speech. For demonstration purposes, this client includes a WeBots simulation that is able to depict a given warehouse and the robot's movement throughout.

How does it work?

First, you enter the desired item using the textbox in the bottom right of the screen; alternatively, you may use the IBM Watson-powered speech to text service by clicking the microphone icon on the left side of the textbox. You can see a conversational interface that allows you to communicate with the Watson assistant to determine the item you have chosen for retrieval. Once the robot knows what to get, it will retrieve it! It's that simple!

What if Robart brings me the wrong item?

We hope to avoid this scenario by offering you a selection of items that match your query in case it is vague or coressponding to multiple items. In the rare cases that situation skipped and Robart assumes a higher level of certainty that it should have, you have a button to report wrong deliveries that the Watson Assistant will learn from and avoid in the future. Furthermore, you may simply ask Robart to return the item.

I want to run the simulation but do not have a powerful computer on premises, what can I do?

The Robart system is designed over a host-client interface. For most of our testing and development, we had a more powerful machine running the robotic simulation that we connected to using the field in the top right corner of the page. Since we got a lot of value and ease of use from this functionality, we decided to bring it to you!

How hard is it to port from simulation to real-world robots?

The Robart system is designed with portability in mind. Each robot runs its own program code in a separate process, so these can be de-centralized and ported to the hardware of each robot you use. The code refers to the the simulation controls abstractly, rather than directly, so porting to new hardware can be as simple as implementing the required interface for a hardware controller.
